# Assets Directory Specification (資產目錄手動歸檔指南)

本目錄專為個人長維護旅遊知識系統（Travel OS）設計，用於手動分類並收納各行程中使用的靜態資源檔案。所有檔案名稱建議加上行程代碼作為前綴（例如：`japan2026_cover.jpg`）。

## 📁 目錄分工與用途描述

### 1. 🖼️ `images/`
- **用途**：放置專案的核心靜態圖標、全局通用 UI 裝飾元素，以及各旅遊計畫的主視覺、Banner 與封面照片（Cover Images）。

### 2. 📸 `photos/`
- **用途**：放置旅途中的實拍照片，主要供未來擴充「相簿卡片（Gallery Block）」或「探店卡片」時手動參照引用。

### 3. 🗺️ `maps/`
- **用途**：收納離線地圖、徒步路線截圖、大眾運輸路網圖，以及自訂的軌跡（如 `.gpx`、`.kml` 檔案導出備份）。

### 4. 📄 `pdf/`
- **用途**：歸檔重要文件的原始檔或 PWA 離線備份，例如航空電子登機證（E-ticket）、海外突發疾病保險單、飯店訂房官方確認信（Confirmation Letter）。

### 5. 🎫 `tickets/`
- **用途**：專門收納景點預約門票、火車票券、新幹線特急券、活動入場 QR Code 截圖，方便手動在 JSON 中綁定相對路徑。

### 6. 📸 `screenshots/`
- **用途**：備份重要網頁資訊、特定店家的特別營業時間（如定休日更動）、或者是當時訂位確認的網頁畫面截圖，防止網頁失效或離線時查無資料。
